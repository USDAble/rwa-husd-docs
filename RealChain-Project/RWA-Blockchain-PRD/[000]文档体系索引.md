- **编号规则**:
  - `[000]`: 文档索引
  - `[001-099]`: 产品愿景和参考文档
  - `[100-199]`: 产品需求文档
  - `[200-299]`: 技术规格文档
  - `[300-399]`: 验收和测试文档

### 文档版本管理

- 使用 Git 进行版本控制
- 每次重大更新增加版本号 (v1.0 → v1.1 → v2.0)
- 在文档开头记录版本号和更新时间

1. 创建或修改文档
2. 更新文档索引 ([000])
3. 更新相关文档的引用关系
4. 提交 Git commit
